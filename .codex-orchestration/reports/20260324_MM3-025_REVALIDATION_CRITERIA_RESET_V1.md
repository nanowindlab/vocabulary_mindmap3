## 1. 기준 재정렬 원칙

- 이번 재검증의 첫 단계는 "비었는가"가 아니라 "왜 비었는가"를 먼저 분류하는 것이다.
- `MM3-020`의 실패 항목은 곧바로 결함으로 고정하지 않고, `MM3-024`의 예외군 정의를 먼저 통과시켜야 한다.
- 재집계는 전체 누락 수를 다시 세는 작업이 아니라, 예외를 제외한 잔여 결함만 다시 세는 작업이다.
- helper 필드는 저장형 결함 집계 대상이 아니라 계산 경로와 반영 경로 확인 대상으로 분리한다.

## 2. 예외군 / 결함군 재분류 기준

### 2.1 source-native 부재 예외

- 원천 데이터가 해당 값을 본질적으로 갖지 않는 경우는 예외군이다.
- 이때는 누락이 아니라 원천 의미의 부재로 본다.
- 단, 예외 인정은 "원래 없다"는 설명만으로는 부족하고, source-native 축이나 대체 facet에서 동일 의미가 재현되지 않음을 함께 보여야 한다.

### 2.2 source-native 대체 예외

- 현재 필드가 비어 있어도 동일 의미가 다른 source-native 축에 존재하면 예외군이다.
- `categories`와 `translations`처럼 thin projection에서 비어 보여도 상위 축 또는 sidecar에서 의미가 살아 있으면 즉시 결함으로 고정하지 않는다.
- 대체 경로가 확인되면 재집계 시 결함 수에서 제외한다.

### 2.3 projection 비대상 예외

- 필드가 해당 화면의 thin projection 범위 밖이면 예외군이다.
- 홈/결과에서 비어 보인다는 이유만으로 상세 계약 위반으로 확장하지 않는다.
- 반대로 상세 필수 필드는 thin projection 논리로 완화하지 않는다.

### 2.4 계산형 예외

- `sense_count`, `has_subwords`, `has_related_forms`는 저장형 누락 검사가 아니라 계산형 반영 검증으로 본다.
- 저장본 미확인을 곧바로 결함으로 세지 않는다.
- 계산 경로가 문서와 runtime 반영에서 열려 있는지 확인하고, 그 자체를 별도 검증 항목으로 둔다.

### 2.5 결함군 판정

- 예외군 어느 쪽에도 들어가지 않고, 화면 계약상 required field이며, 대체 경로도 없는 경우만 결함군이다.
- `entry.pos`처럼 기본 식별 축에 가까운 필드는 예외 인정 범위가 가장 좁다.
- `categories`와 `translations`는 source-native 대체 경로가 확인되지 않을 때만 결함군으로 남긴다.

## 4. 재검증 기준값

- canonical count는 `53,480`으로 고정한다.
- 홈 `categories`의 재검증 기준값은 "예외 제외 후 잔여 결함 `0`"이다.
- 결과 `translations`의 재검증 기준값은 "예외 제외 후 잔여 결함 `0`"이다.
- 홈 `pos`의 재검증 기준값은 "잔여 결함 `0`"이며, 예외 인정은 문서화된 source-native 근거가 있을 때만 허용한다.
- `sense_count / has_subwords / has_related_forms`는 `0건`이라는 저장형 기준을 적용하지 않고, 계산 경로 존재와 반영 일치 여부로 판정한다.
- 다음 패킷에서 허용되는 상태는 `예외군 확정 + 잔여 결함 0 + canonical count 유지`뿐이다.

## 5. 다음 검증 패킷 진입 조건

1. `categories`, `translations`, `pos` 각각에 대해 예외군 후보와 결함군 후보가 문서상 분리되어 있어야 한다.
2. 예외로 제외된 수와 잔여 결함 수가 각각 따로 집계되어 있어야 한다.
3. `sense_count / has_subwords / has_related_forms`는 저장형 누락 검사에서 분리되어 계산 경로 검증 대상으로만 남아 있어야 한다.
4. canonical count `53,480`이 유지되어야 한다.
5. 잔여 결함이 하나라도 남아 있으면 다음 검증 패킷은 `NO`다.

## 6. 운영용 판정 순서

2. 아니면 source-native 대체 경로가 있는지 본다.
3. 아니면 projection 비대상인지 본다.
4. 아니면 계산형 항목인지 본다.
5. 마지막에만 결함군으로 남길지를 판정한다.

## 7. Reflection

- 이번 재정렬의 핵심은 누락 수를 줄이는 것이 아니라, 결함으로 세면 안 되는 항목을 먼저 빼는 데 있다.
- `categories`와 `translations`는 thin projection에서 보이는 빈칸과 실제 결함이 섞이기 쉬우므로, 재집계 전에 예외 판정 축을 먼저 고정해야 한다.
- `pos`는 예외 여지가 좁기 때문에 원칙적으로 결함군에 남기되, 비표준 source class가 확인될 때만 재검토한다.
- helper 필드를 저장형 결함으로 계속 세면 검증 모델이 과잉 실패로 기울기 때문에, 이번 패킷에서는 계산 경로 검증으로 분리하는 것이 맞다.
