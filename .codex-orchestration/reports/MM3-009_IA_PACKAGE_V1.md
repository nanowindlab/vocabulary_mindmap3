## 1. 목적과 범위

- 본 문서는 `word-first + dual category + sense core`를 기준으로 한 화면 단위 IA package다.
- 핵심 원칙은 `다중 진입점 + 다층 레이어 + orthogonal filter`다.
- 구조 뼈대에서는 `mindmap2`의 `related_vocab`, `crosslink`를 사용하지 않는다.
- 이 문서는 구현 지시서가 아니라, 홈/검색·결과/상세/표현층/필터 바의 정보 구조와 이동 흐름을 결정하기 위한 IA 문서다.

## 2. 작업 -> 검증 -> learner 포함 3인 전문가 비판 검토 -> 개선 -> 재검증

### 작업

- PM 구조 옵션 보고서의 결론을 기준점으로 삼아, 기본 진입을 `word-first`로 두고 내부 구조를 `dual category`와 `sense core`로 분해했다.
- `MM3-012`의 마인드맵 상호작용 규칙을 기준으로, IA 안에서 캔버스/상태층이 별도 모드가 아니라 공통 탐색 레이어로 읽히는지 다시 점검했다.
- source discovery와 scenario rewrite 보고서의 역할 분담을 다시 확인해, 화면별 목적과 노출 레이어를 분리했다.

### 검증

- `DATA_ARCHITECTURE_V1.md`와 source discovery 보고서를 대조해 `entry`, `sense`, `categories`, `subwords`, `translations`, `related_forms`의 역할이 섞이지 않도록 점검했다.
- `학습난이도`, `TOPIK`, `품사`, `외국어 종류`는 taxonomy 축이 아니라 orthogonal filter라는 전제를 다시 확인했다.
- 홈/검색/결과/상세/표현층 전 구간에서 마인드맵이 화면을 대체하지 않고, 현재 표면 위에서 중심축을 바꾸는 상태층으로만 동작해야 한다는 점을 다시 확인했다.

### learner 포함 3인 전문가 비판 검토

- learner 관점: 첫 진입은 가장 짧은 lookup 경로여야 하므로 홈의 기본 동선은 검색 중심이어야 한다.
- source 관점: `의미 범주`와 `주제 및 상황 범주`는 서로 다른 진입 의도를 대표하므로 같은 레이어로 합치면 안 된다.
- PM 관점: `sense`를 중심 레이어로 두지 않으면 다의어, 번역 정합성, 예문 비교가 모두 약해진다.

### 개선

- 홈, 검색/결과, 상세, 표현층, 필터 바를 각각 독립 화면/섹션으로 분리했다.
- 항상 보이는 요소와 2차 노출 요소를 구분해 learner 부담을 낮추도록 재배치했다.
- IA 기준에서 decision 단계로 되돌아가야 하는 조건을 별도 항목으로 고정했다.
- 마인드맵 진입/동기화/이탈을 화면 이동 흐름과 분리해, IA가 상태 전이 규칙까지 함께 읽히도록 보강했다.

### 재검증

- 본 패키지가 `word-first + dual category + sense core`와 충돌하지 않는지 다시 확인했다.
- `mindmap2`의 graph 관성을 구조 뼈대에서 제외하는 것이 source 정합성과 learner flow에 더 안전하다는 결론을 유지했다.

## 3. IA 한 줄 정의

- 기본 진입은 `word-first`다.
- 의미 구조는 `sense core`가 담당한다.
- category는 `의미 범주`와 `주제 및 상황 범주`의 `dual category`로 병렬 제공한다.
- 표현은 본체 taxonomy와 분리된 `표현층`으로 2차 노출한다.
- 필터는 범주를 대체하지 않는 orthogonal control로 상시 유지한다.

## 4. 화면 구조 총괄

### 4.1 홈

- 목적: 가장 빠른 lookup과 병렬 진입점을 함께 제공하는 허브다.
- 구성:
  - 표제어 검색창
  - `의미 범주` 진입
  - `주제 및 상황 범주` 진입
  - `표현층` 보조 진입
  - 상단 고정 필터 바
  - 대표 탐색 힌트 또는 최근/추천 맥락
- 해석:
  - 홈은 하나의 트리로 사용자를 강제하지 않는다.
  - 검색이 기본값이지만, category 기반 학습 경로도 동등한 출발점으로 제공한다.

### 4.2 검색/결과

- 목적: 입력 의도와 후보군을 가장 짧은 경로로 정렬하는 화면이다.
- 구성:
  - 검색 입력 유지
  - 결과 목록
  - 결과별 `sense` 요약
  - 관련 category 태그
  - 필터 바
  - 결과가 부족할 때의 대체 진입 안내
- 해석:
  - 검색/결과는 단순 목록이 아니라 `word -> sense -> category`로 재분기되는 관문이다.
  - 결과 화면에서 사용자는 같은 표제어의 다른 의미와 범주를 비교한다.

### 4.3 상세

- 목적: 하나의 표제어를 `sense core` 기준으로 해석하고 학습하는 화면이다.
- 구성:
  - 표제어 기본 정보
  - `sense` 카드 목록
  - 각 sense의 정의, 번역, 예문
  - 문형, 발음, 주석, 멀티미디어 같은 보조 정보
  - 관련어와 관련형
  - 표현층으로의 하위 확장 진입
- 해석:
  - 상세는 entry 전체보다 sense별 비교와 해석에 초점을 둔다.
  - 번역과 예문은 entry 단위가 아니라 sense 단위 정합성을 우선한다.

### 4.4 표현층

- 목적: 관용구, 속담, 고정 표현, subword 성격의 묶음을 별도 맥락으로 제공하는 화면 또는 섹션이다.
- 구성:
  - 표현류 목록
  - 표현별 의미/용례
  - 연결된 entry 또는 sense 참조
  - 표현층 내부 필터 또는 탐색 단서
- 해석:
  - 표현층은 본체 taxonomy와 같은 층위가 아니다.
  - 독립 출발점이 될 수 있지만, 홈의 1차 시각 노출은 낮춘다.

### 4.5 홈 상단 고정 필터 바

- 목적: 진입점을 대체하지 않으면서 탐색 범위를 줄이는 상시 제어판이다.
- 구성:
  - `학습난이도`
  - `TOPIK`
  - `품사`
  - `외국어 종류`
- 해석:
  - 필터는 축이 아니라 조건이다.
  - 상단에 고정하되 시각 밀도는 낮게 유지한다.
  - 홈과 결과 화면에서 공통으로 보여주되, 기본 상태는 요약 중심이어야 한다.

### 4.6 마인드맵 캔버스/상태층

- 목적: 마인드맵을 별도 화면이 아니라 홈/검색/결과/상세/표현층 위에서 함께 작동하는 공통 탐색 캔버스로 다루는 것이다.
- 캔버스 규칙:
  - 홈은 기본 landing이고, 마인드맵은 그 위에서 열리는 상태층이다.
  - 마인드맵은 현재 화면의 중심축을 시각화하지만 화면 자체를 바꾸지는 않는다.
  - `word`, `sense`, `category`, `filter`는 같은 캔버스 위에서 서로 독립적인 축으로 유지한다.
- 상태층 규칙:
  - `selected_word`, `selected_sense`, `selected_category_meaning`, `selected_category_situation`, `active_filters`, `current_mode`를 하나의 복합 값으로 합치지 않는다.
  - `current_surface`는 현재 표면을, `projection_depth`는 노출 깊이를 나타낸다.
  - 상태층은 선택 결과를 반영하지만 원본 구조를 덮어쓰지 않는다.

## 5. 화면 간 이동 흐름

1. 홈에서 표제어 검색 또는 category 진입을 시작한다.
2. 검색어가 있으면 검색/결과 화면에서 후보군을 비교한다.
3. 결과에서 원하는 표제어를 선택하면 상세로 들어간다.
4. 상세에서 sense를 선택해 정의, 예문, 번역, 관계어를 확인한다.
5. 필요할 때 표현층으로 내려가 관용구/속담/고정 표현을 본다.
6. category 학습 의도가 강하면 홈 또는 결과에서 `의미 범주`와 `주제 및 상황 범주`로 되돌아간다.

### 마인드맵 진입 요점

- 마인드맵은 홈에서 가장 자연스럽게 열리지만, 검색/결과/상세/표현층에서도 같은 규칙으로 진입할 수 있다.
- 진입 시 기준은 항상 현재 표면의 선택 상태이며, 별도 모드 전환이 선행되지 않는다.
- 홈 진입은 허브형 진입, 상세 진입은 의미 확장형 진입, 표현층 진입은 보조 맥락 진입으로 읽는다.

### 마인드맵 동기화 요점

- 동기화 우선순위는 `word -> sense -> category -> filter`다.
- `word`가 바뀌면 `sense`를 먼저 재선정하고, 그다음 `category`와 `filter`를 맞춘다.
- `sense`가 바뀌면 번역, 예문, 관련어, 문형은 해당 sense 기준으로 즉시 따라간다.
- `category`는 `의미 범주`와 `주제 및 상황 범주`를 독립 축으로 유지한 채 동기화한다.

### 마인드맵 이탈 요점

- 탐색이 하나의 word, sense, category로 수렴하지 않으면 홈으로 이탈한다.
- 이탈 시 현재 필터와 마지막 선택 상태는 다음 진입을 돕는 수준으로만 얇게 남긴다.
- 이탈은 상태 초기화가 아니라 탐색 초점의 재배치다.

## 6. 항상 보이는 요소와 2차 노출 요소

### 항상 보이는 요소

- 표제어 검색 진입
- 홈 상단 고정 필터 바
- `의미 범주`와 `주제 및 상황 범주`의 병렬 진입
- 검색/결과의 결과 목록과 sense 요약
- 상세의 sense 카드 핵심 정보

### 펼침 또는 2차 노출 요소

- 표현층 전체
- sense의 보조 메타 정보
- 문형, 발음, 주석, 멀티미디어의 상세 확장
- 관련어와 관련형의 세부 묶음
- 필터의 세부 조합

## 7. learner 부담 완화를 위한 노출 강도 원칙

- 첫 화면은 설명보다 발견을 우선한다.
- 진입점은 많되 동시에 강하게 밀지 않는다.
- `word-first`를 기본값으로 유지하고, category와 표현은 병렬 또는 후순위로 둔다.
- `sense`는 중심이지만, 모든 정보를 한 번에 펼치지 않는다.
- 필터는 고정하되 눈에 띄는 주인공이 되지 않게 낮은 대비로 둔다.
- 표현층은 학습 의도가 분명할 때만 강하게 보이도록 2차 노출로 유지한다.
- 같은 화면에서 선택지가 많아질수록 기본 상태의 정보량은 줄여야 한다.

## 8. `dual category` 처리 원칙

- `의미 범주`는 개념 축이다.
- `주제 및 상황 범주`는 사용 장면 축이다.
- 둘은 같은 UI 덩어리 안에 있더라도 역할은 다르다.
- 두 범주는 병렬로 보이되, 하나가 다른 하나를 설명하지 않는다.
- 사용자는 `이 단어가 무엇을 뜻하는가`와 `어떤 상황에서 쓰는가`를 분리해서 확인할 수 있어야 한다.

## 9. 상세의 `sense core` 원칙

- 상세의 기본 단위는 entry가 아니라 sense다.
- sense에는 정의, 번역, 예문, 관계어가 함께 붙는다.
- entry는 표제어의 외피이고, sense는 학습 단위다.
- 다의어 처리, 번역 정합성, 예문 비교는 sense에서만 안정적으로 수행된다.

## 10. IA 기준에서 다시 decision 단계로 돌아가야 하는 조건

- 사용자가 검색보다 category 진입을 더 자주 기본값으로 쓰는 징후가 확인될 때
- `의미 범주`와 `주제 및 상황 범주` 중 하나가 사실상 보조 기능으로만 소비될 때
- sense 중심 상세가 초급 학습자에게 과도하게 무거운 것으로 확인될 때
- 표현층이 독립 런치패드 수준으로 커져 본체 구조를 다시 나눠야 할 때
- 필터가 category보다 더 자주 핵심 진입점처럼 사용될 때
- 이후 검증에서 source 구조와 실제 learner flow가 현재 IA와 다르게 드러날 때
- 마인드맵 상태층이 화면 전환을 대체하는 별도 모드처럼 소비될 때

## 11. Reflection

- 이번 IA 패키지는 구조를 더 복잡하게 만드는 작업이 아니라, 진입점과 레이어의 우선순위를 분리하는 작업이었다.
- `word-first + dual category + sense core`는 화면 수를 늘리는 원칙이 아니라, 같은 표제어를 서로 다른 읽기 방식으로 안정적으로 보여주기 위한 기준이다.
- `mindmap2`의 연결망 관성을 버리고 source-native 구조로 정리한 것이 이번 패키지의 가장 중요한 결론이다.
- 남은 불확실성은 구조의 옳고 그름보다 실제 learner uptake 비율이므로, 다음 decision에서는 사용 패턴 증거를 기준으로 다시 판정해야 한다.
