# MM3-012_MINDMAP_INTERACTION_SPEC_V1

## Revision Meta

- Current Revision: `R1`
- Last Updated: `2026-03-23`
- Last Updated By: `MINDMAP_AGENT`

## 1. 목적과 범위

- 본 문서는 `word-first + dual category + sense core`를 전제로, MM3 구조 안에서 마인드맵이 어떻게 진입, 동기화, 이탈하는지 정의하는 상호작용 명세다.
- 핵심 원칙은 `다중 진입점 + 다층 레이어 + orthogonal filter`다.
- 구현 지시나 코드 변경은 포함하지 않는다.
- 판단 기준은 홈 / 검색 / 결과 / 상세 / 표현층 / 필터 바의 실제 사용자 이동을 일관되게 설명할 수 있는가에 둔다.

## 2. 작업 -> 검증 -> learner 포함 3인 전문가 비판 검토 -> 개선 -> 재검증

### 작업

- IA 패키지와 runtime contract를 기준으로, 마인드맵을 별도 기능이 아니라 MM3 전체에 걸친 탐색 구조로 재정의했다.
- source discovery와 scenario rewrite 보고서를 대조해, `entry`, `sense`, `subword`, `translation`, `category`, `filter`의 역할이 섞이지 않도록 정리했다.

### 검증

- `word-first`가 기본 진입이라는 점과 `sense core`가 상세 해석의 중심이라는 점을 다시 확인했다.
- `의미 범주`와 `주제 및 상황 범주`는 병렬 축이며, 필터는 이를 대체하지 않는 orthogonal control이라는 전제를 재확인했다.

### learner 포함 3인 전문가 비판 검토

- learner 관점: 첫 진입은 빠른 확인과 재탐색이 쉬워야 하므로, 홈은 그대로 남고 마인드맵은 필요할 때 열리는 구조가 더 자연스럽다.
- source 관점: 의미와 상황 범주를 같은 그래프 규칙으로 묶으면 source-native 분리가 무너진다.
- PM 관점: MM2의 기본화면 관성을 유지하면 `word-first + dual category + sense core`의 우선순위가 흐려진다.

### 개선

- 마인드맵을 기본 화면이 아니라, 홈을 포함한 여러 화면 위에서 재사용되는 탐색 상태층으로 정리했다.
- 진입, 동기화, 이탈을 각각 별도 규칙으로 나누고, 상태 단위를 최소 집합으로 고정했다.
- MM2 기본화면 모델을 유지하지 않는 결론을 명시했다.

### 재검증

- 홈 / 검색 / 결과 / 상세 / 표현층 각각에서 마인드맵이 어떤 방식으로 열리고 닫히는지 다시 점검했다.
- 선택 단위가 바뀔 때 `word`, `sense`, `category`, `filter`, `mode`가 어떤 우선순위로 갱신되는지 문장 수준으로 재정리했다.

## 3. 결론 요약

- MM3의 기본 landing은 홈이다.
- 마인드맵은 기본 화면이 아니라, 홈과 후속 화면 전체에 공통 적용되는 구조화된 탐색 레이어다.
- 따라서 마인드맵은 별도 앱 모드라기보다, 화면 전환을 가로지르는 상태 표현 방식에 가깝다.
- 홈은 진입 허브이고, 마인드맵은 그 허브를 포함해 검색 / 결과 / 상세 / 표현층을 연결하는 공통 맥락이다.

## 4. 기본 화면, 별도 모드, 홈의 관계

### 기본 화면 여부

- 마인드맵은 기본 화면이 아니다.
- 기본 화면은 홈이다.
- 이유는 `word-first`에서 첫 행동은 구조 탐색보다 lookup이 먼저이기 때문이다.

### 별도 모드 여부

- 마인드맵은 완전히 독립된 별도 모드도 아니다.
- 홈과 검색 / 결과 / 상세 / 표현층 위에 얹히는 공통 탐색 레이어다.
- 사용자는 화면 자체를 바꾸는 것이 아니라, 현재 화면의 해석 방식과 초점을 바꾼다.

### 홈과의 관계

- 홈은 기본 landing이고, 마인드맵은 홈 안에서 가장 먼저 보일 수 있는 구조 요약이자 탐색 허브다.
- 홈의 역할은 시작점 제공이고, 마인드맵의 역할은 시작점들 사이의 관계를 보여 주는 것이다.
- 즉 홈은 시작 위치, 마인드맵은 시작 위치를 선택하고 이어 붙이는 방식이다.

## 5. 홈 / 검색 / 결과 / 상세 / 표현층에서 마인드맵으로 들어가는 방식

### 홈에서 진입

- 홈은 기본 landing이다.
- 홈에서 마인드맵은 검색창, `의미 범주`, `주제 및 상황 범주`, `표현층` 진입을 함께 묶어 보여 주는 허브형 진입점으로 열린다.
- 홈에서 진입한 마인드맵은 아직 특정 word가 고정되지 않은 상태일 수 있다.

### 검색에서 진입

- 검색은 `word-first` 진입의 출발점이다.
- 검색어가 입력되면 마인드맵은 그 단어의 후보군과 sense 분해 상태를 보여 주는 방향으로 열린다.
- 검색에서의 마인드맵은 전체 구조 탐색보다 후보 정렬과 재분기 기능이 강하다.

### 결과에서 진입

- 결과는 `word -> sense -> category`로 재분기되는 관문이다.
- 결과에서 특정 entry를 선택하면 마인드맵은 그 entry를 중심으로 열리고, sense 요약과 category 경로가 함께 갱신된다.
- 결과에서 마인드맵은 비교와 선택을 돕는 재분기 화면이다.

### 상세에서 진입

- 상세는 sense core를 기준으로 해석하는 화면이다.
- 상세에서 마인드맵은 선택된 entry와 선택된 sense를 중심으로 열리며, 번역, 예문, 관련어, 표현층 참조가 함께 동기화된다.
- 상세에서의 마인드맵은 구조 탐색보다 의미 확장의 역할이 강하다.

### 표현층에서 진입

- 표현층은 subword, 관용구, 속담, 고정 표현을 다루는 별도 레이어다.
- 표현층에서 마인드맵은 parent entry 또는 linked sense를 앵커로 삼아 열리며, 표현 간 관계와 본체 어휘 연결을 보여 준다.
- 표현층 진입은 본체 word를 대체하지 않고, 본체에 연결된 보조 맥락으로 들어가는 방식이다.

## 6. 선택 변화에 따른 동기화 규칙

### 단어를 선택했을 때

- 새로운 word가 선택되면 마인드맵의 중심은 해당 entry로 이동한다.
- 기존 sense 선택은 같은 entry 안에서는 유지할 수 있지만, 다른 entry로 넘어가면 재선정이 필요하다.
- category와 filters는 가능하면 유지하되, 새 word와 충돌하면 재계산된다.

### sense를 선택했을 때

- sense 선택은 동일 word 안에서 의미 초점을 바꾸는 행위다.
- word는 유지되고, selected sense만 바뀐다.
- 번역, 예문, 관련어, 문형은 선택된 sense에 맞춰 즉시 동기화된다.

### category를 선택했을 때

- `의미 범주`와 `주제 및 상황 범주`는 각각 독립적으로 선택된다.
- 한 category를 선택해도 다른 category 축은 자동 삭제하지 않는다.
- 다만 같은 축에서 더 구체적인 category를 고르면 상위 선택은 하위 선택으로 접힌다.

### 표현을 선택했을 때

- 표현 선택은 subword 계열을 우선하는 선택이다.
- 표현은 parent word 또는 linked sense를 그대로 보존하면서 보조 앵커만 추가한다.
- 표현을 선택했다고 해서 본체 word의 sense 구조가 사라지면 안 된다.

### 검색 조건이 바뀌었을 때

- 검색어가 바뀌면 현재 마인드맵은 새 후보 집합으로 재정렬된다.
- 필터는 원칙적으로 유지한다.
- 단, 새 검색어와 필터가 충돌해 결과가 비는 경우에는 필터 유지보다 복구 가능한 방향으로 완화한다.

## 7. 마인드맵에서 다시 나오는 방식

### 상세로 복귀

- 마인드맵에서 특정 sense를 더 깊게 선택하면 상세로 복귀한다.
- 복귀 시 상세는 선택된 sense를 중심으로 열리고, entry 전체는 배경 컨텍스트로 남는다.

### 검색으로 복귀

- word를 다시 바꾸거나 후보를 재검색하면 검색 화면으로 복귀한다.
- 검색 복귀는 같은 탐색 맥락을 유지한 채 후보군만 교체하는 방식이다.

### 범주로 복귀

- category 중심 탐색이 목적이면, 마인드맵에서 `의미 범주` 또는 `주제 및 상황 범주` 뷰로 복귀한다.
- 복귀 시 word보다 범주 집합이 우선한다.

### 표현층으로 복귀

- 표현 관련 선택이 이어지면 표현층으로 복귀한다.
- 이때 parent entry와 linked sense는 유지하고, 표현 후보만 바뀐다.

### 홈으로 이탈

- 현재 탐색이 더 이상 하나의 word, sense, category로 수렴하지 않으면 홈으로 이탈한다.
- 홈으로 나갈 때는 현재 필터와 마지막 선택 상태를 완전히 삭제하지 않고, 다음 진입을 돕는 수준으로만 남긴다.

## 8. 마인드맵 상태 단위

### 필수 상태

- `selected_word`: 현재 중심 표제어
- `selected_sense`: 현재 중심 의미 단위
- `selected_category_meaning`: `의미 범주` 축 선택 상태
- `selected_category_situation`: `주제 및 상황 범주` 축 선택 상태
- `active_filters`: `학습난이도`, `TOPIK`, `품사`, `외국어 종류`
- `current_mode`: browse, search, compare, detail, expression 같은 현재 상호작용 모드

### 보조 상태

- `current_surface`: 홈, 검색, 결과, 상세, 표현층 중 현재 표면
- `navigation_origin`: 어디에서 마인드맵으로 들어왔는지
- `projection_depth`: 요약 상태인지 rich detail인지

### 상태 해석 원칙

- `word`는 가장 바깥의 기준점이다.
- `sense`는 의미 해석의 핵심이다.
- `category`는 해석 축이고, `filter`는 조건 축이다.
- `mode`는 현재 사용자가 무엇을 하려는지 나타내는 상호작용 상태다.
- 이 상태들은 하나로 합치지 말고 서로 독립적으로 유지해야 한다.

## 9. 동기화 우선순위

1. 선택된 `word`를 먼저 맞춘다.
2. 그 word에 종속된 `sense`를 맞춘다.
3. 그다음 `category` 축을 맞춘다.
4. 마지막에 `filters`를 맞춘다.

- 이 순서를 뒤집으면 범주와 조건이 word보다 앞서게 되어 `word-first` 원칙이 흔들린다.
- 반대로 sense를 건너뛰면 다의어와 번역 정합성이 약해진다.
- 따라서 마인드맵 동기화는 항상 `word -> sense -> category -> filter` 순으로 이해하는 편이 안전하다.

## 10. MM2 기본화면 모델 유지 여부

- MM2 기본화면 모델은 유지하지 않는다.
- 이유는 MM2의 기본화면 관성, graph 중심 관성, 연결망 강조가 MM3의 source-native 구조와 맞지 않기 때문이다.
- MM3는 `word-first + dual category + sense core`를 전제로 하므로, 기본 화면을 그래프 중심으로 두면 entry와 sense의 위계가 흐려진다.
- 다만 MM2에서 참고할 수 있는 것은 화면 리듬과 탐색 습관 정도이며, 구조 뼈대 자체는 유지하지 않는다.

## 11. Reflection

- 이번 명세의 핵심은 마인드맵을 더 큰 기능으로 만드는 것이 아니라, MM3의 여러 화면이 같은 해석 규칙을 공유하게 만드는 것이었다.
- `word-first`를 기본값으로 유지하면서도 `dual category`와 `sense core`를 잃지 않으려면, 마인드맵은 독립 화면이 아니라 상태층이어야 한다.
- 홈은 시작점이고, 마인드맵은 시작점들 사이의 관계를 바꾸는 방식이다.
- 다음 검증에서는 이 명세가 실제 learner 흐름에서 화면 수를 늘리지 않으면서도 재탐색 비용을 낮추는지 확인해야 한다.
